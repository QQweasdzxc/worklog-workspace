# 🪶 Zhuge AI OS

by Mr. KM

Zhuge AI OS 不是一套功能很多的系統，而是一位會陪著使用者完成工作的 AI 工作夥伴。

第一版產品主線很單純：

```text
一句自然語言
↓
Mr. KM 理解
↓
使用者確認
↓
寫入工時月曆
↓
月底匯出 ECP
```

## Product Charter

- 每一次互動，都讓 Mr. KM 更懂你的工作。
- 每一次學習，都讓下一次工時更容易完成。
- AI 負責理解與建議，最後決定權永遠在使用者。
- 文件不是目的；文件只是讓 Mr. KM 學習工作的教材。

產品原則：[Teach Mr. KM, Not Manage Files](docs/product-principles.md)

## Why Mr. KM?

Mr. KM 不是一個功能名稱，而是 Zhuge AI OS 的產品人格。

KM 代表：

- **Knowledge Mate**：陪著使用者一起工作的知識夥伴。
- **Knowledge Mentor**：隨著每一次互動，逐步理解使用者工作方式的指引者。
- **Knowledge Companion**：不要求使用者適應系統，而是努力配合使用者的工作節奏。

Mr. KM 不追求知道全世界。

他追求的是：

> 越來越懂這位使用者的工作，並讓下一次工時更容易完成。

## 目前能力

- 自然語言建立工時，確認後寫入正式工時月曆。
- Web / Mobile / Chrome Extension 共用同一份 Conversation Core。
- 工時、我的工作、ECP 設定、Conversation 與藏書閣走 Supabase Cloud Sync。
- 🪶 Mr. KM 建議以「我的工作」為來源，協助使用者更快加入工時。
- 藏書閣用來教 Mr. KM 學會新的工作，而不是管理檔案。
- P5.3 Work Memory：文件整理後可經使用者確認加入「我的工作」，讓 KM 建議逐步更貼近工時。
- P5.3 Smart Merge：Mr. KM 會主動發現相近工作，提出整理建議，再由使用者決定是否接受。
- P5.3 Companion Card：每一項「我的工作」都會說明 Mr. KM 從哪裡學會、最近何時陪你完成、之後能怎麼幫你。
- P5.3 IA Simplification：「我的工作」收斂到設定中的工作管理中心，不再作為重複入口。
- P5.3 Companion Card UX Refinement：「我的工作」第一層只保留來源與編輯；AI 整理建議只在 Mr. KM 發現有價值時出現。

## 🪶 Mr. KM 成長日誌

### 2026/07

今天，
我開始學會把 SOP 與文件整理成可用於工時建議的工作。

以前，
我只能保存文件。

現在，
我開始知道主人每天可能會做哪些工作，並把它們連回「我的工作」與工時。

接下來，
我開始建立 Work Memory。

我不只是記住你的文件，
我希望記住的是你的工作。

現在，
我也開始學會整理你的工作。

如果我發現兩項工作很相近，
我會先提出建議，
最後仍由你決定。

我也開始學會用更像夥伴的方式介紹「我的工作」。

不是告訴你欄位，
而是告訴你：

我從哪裡學會這項工作，
以及之後可以怎麼陪你完成它。

現在，
我也開始學會不要把所有功能都變成按鈕。

如果我發現兩項工作很像，
我會主動提醒你；
如果沒有值得整理的地方，
我就安靜陪你把工時完成。

現在，
我也開始把入口收斂得更清楚。

「我的工作」不是每天要打開的功能，
而是我理解你工作的基礎。

它會留在設定裡，
成為之後 KM 建議與工時建立的主要來源。
