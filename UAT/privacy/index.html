隱私權政策

生效日期：2026 年 7 月 17 日

Zhuge AI OS（以下稱「本服務」）重視使用者對資料的控制。本政策說明本服務蒐集、使用、儲存與刪除資料的方式，包含透過 Google API 取得的使用者資料。

1. 我們處理的資料

  • Google 帳號基本資料：Google 帳號識別碼、姓名、電子郵件與頭像，用於登入、建立帳號及跨裝置同步。
  • Google Drive 資料：在使用者主動啟用 Drive 功能後，讀取使用者指定資料夾內的檔名、File ID、MIME type、修改時間及支援文件的內容。
  • 工作資料：工時、工作身分、「我的工作」、對話、使用者確認的知識內容與必要的操作紀錄。

2. 使用目的

資料僅用於提供使用者可見的功能：Google 登入、同步工作資料、讀取使用者指定的工作文件、建立工作知識、改善 Mr. KM 的工時建議，以及維護服務安全與可靠性。

我們不販售 Google 使用者資料，不將資料用於廣告，也不將資料用於與 Zhuge AI OS 工作協助功能無關的用途。

3. Google API Limited Use

Zhuge AI OS 對 Google API 所取得資料的使用與傳輸，遵守 Google API Services User Data Policy，包含 Limited Use 要求。

本服務目前僅要求基本登入權限與 drive.readonly。Drive 權限為唯讀；本服務不會透過此權限修改或刪除使用者的 Google Drive 檔案。

4. 儲存與第三方服務

帳號、工時、工作記憶與使用者選擇保存的文件衍生內容可能儲存於 Supabase 所提供的 Auth、Database 與 Storage。Google OAuth 流程由 Google 與 Supabase Auth 完成。

只有提供服務所必要的資料會被傳送至這些處理服務。我們不會在 Console、公開記錄或錯誤訊息中輸出 OAuth token 或完整私人文件內容。

5. 保留與刪除

資料會保留至使用者在產品中刪除、撤銷授權、要求刪除帳號，或服務不再需要該資料時。使用者可依支援與刪除說明提出帳號與 Google 衍生資料刪除要求。

6. 撤銷 Google 授權

使用者可隨時前往 Google 帳號的第三方連線頁撤銷 Zhuge AI OS 的存取權。撤銷後，本服務將無法繼續讀取 Google Drive；已由使用者選擇保存於 Zhuge AI OS 的資料，仍可另行要求刪除。

7. 安全

本服務採用 HTTPS、OAuth 2.0、PKCE 與存取控制，以降低未授權存取風險。任何網路服務都無法保證絕對安全；若發現安全事件，我們會依事件程度採取限制、撤銷與通知措施。

8. 聯絡方式

隱私權、資料存取或刪除問題，請寄信至 qq.1025@gmail.com。

English summary

Zhuge AI OS uses Google identity data for sign-in and, only after explicit authorization, read-only Google Drive data for user-facing work knowledge and WorkLog assistance. It does not sell Google user data, use it for advertising, or modify Drive files. Users may revoke access at any time and request deletion through the support page.
